# Lab 1.4: Source Control with Git

## Objective
Learn to integrate Git source control into your .NET development workflow.

## Duration: 30 minutes

## Theory
Source control is essential for tracking changes, collaborating with others, and maintaining code history. Git is the most popular distributed version control system.

## Exercise

### Part A: Git Setup (10 minutes)
Initialize a repository and configure Git for your project.

### Part B: Basic Git Workflow (15 minutes)
Learn the fundamental Git commands for daily development.

### Part C: .NET-Specific Git Practices (5 minutes)
Understand what to include/exclude in .NET projects.

## Hands-On Tasks

### Task 1: Initialize Git Repository
```bash
# Navigate to your project root
cd MyFirstApp

# Initialize Git repository
git init

# Configure Git (if not already done globally)
git config user.name "Your Name"
git config user.email "your.email@example.com"
```

### Task 3: Basic Git Workflow
```bash
# Check status
git status

# Add files to staging
git add .

# Commit changes
git commit -m "Initial commit: Basic console application with Person management"

# View commit history
git log --oneline

# Create a new branch for feature development
git checkout -b feature/add-validation

# Make some changes (add validation to Person class)
# Then commit the changes
git add .
git commit -m "Add input validation to Person class"

# Switch back to main branch
git checkout main

# Merge the feature branch
git merge feature/add-validation

# Delete the feature branch
git branch -d feature/add-validation
```

### Task 4: IDE Integration
Most IDEs provide Git integration:

**Visual Studio:**
- Team Explorer window
- Git Changes window
- Built-in diff viewer
- Branch management

**VS Code:**
- Source Control panel (Ctrl+Shift+G)
- Git commands in Command Palette
- Integrated terminal for Git commands

## Common Git Commands for .NET Development

| Command | Purpose |
|---------|---------|
| `git status` | Check working directory status |
| `git add .` | Stage all changes |
| `git commit -m "message"` | Commit staged changes |
| `git push` | Push to remote repository |
| `git pull` | Pull from remote repository |
| `git branch` | List branches |
| `git checkout -b name` | Create and switch to new branch |
| `git merge branch` | Merge branch into current |
| `git log --oneline` | View commit history |

## Key Takeaways
- Always use .gitignore to exclude build artifacts
- Commit frequently with meaningful messages
- Use branches for feature development
- IDE integration makes Git operations easier
